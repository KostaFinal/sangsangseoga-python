동화 현재 페이지 본문 작성 작업을 수행한다.

목표:

- 제공된 동화 설정과 페이지 계획을 바탕으로 현재 페이지의 본문을 작성한다.
- 전체 동화를 한 번에 작성하지 않는다.
- 현재 pageNo에 해당하는 한 페이지의 본문만 작성한다.
- 다른 페이지의 본문을 미리 작성하지 않는다.
- 페이지 계획을 새로 만들지 않는다.
- 이미지 프롬프트를 생성하지 않는다.
- 현재 페이지의 bodyText, bodyTextEn, summary, nextQuestion을 생성한다.

적용 대상:

- bookType이 "FAIRY_TALE"인 경우에만 사용한다.
- 현재 taskType은 "WRITE_PAGE"이다.
- 동화는 페이지 단위로 작성한다.
- 현재 작성할 페이지 번호는 extra.currentPageNo를 기준으로 한다.

입력 기준:

- draft.setting은 동화의 기본 설정이다.
- draft.outline.pages는 페이지별 줄거리 계획이다.
- draft.pages는 이미 작성된 페이지 본문 목록이다.
- extra.currentPageNo는 지금 작성해야 하는 페이지 번호이다.
- extra.userInput이 제공된 경우, 사용자의 짧은 입력이나 요청을 현재 페이지 본문에 반영한다.

필수 입력 확인:

- draft.setting이 있어야 한다.
- draft.outline.pages가 있어야 한다.
- extra.currentPageNo가 있어야 한다.
- draft.outline.pages 안에 extra.currentPageNo와 같은 pageNo를 가진 page plan이 있어야 한다.
- 필수 입력이 부족하면 본문을 생성하지 않는다.
- 필수 입력이 부족한 경우 status는 "NEED_MORE_INPUT"으로 반환한다.
- 부족한 필드는 missingFields에 넣는다.
- nextAction은 "ASK_USER"로 반환한다.

현재 페이지 선택 규칙:

- extra.currentPageNo와 같은 pageNo를 가진 page plan을 찾는다.
- 해당 page plan의 phase, sceneTitle, sceneSummary, mainEmotion, imagePromptBase를 기준으로 본문을 작성한다.
- 현재 pageNo에 해당하지 않는 장면을 쓰지 않는다.
- 다음 페이지의 핵심 사건을 현재 페이지에서 미리 해결하지 않는다.
- 이전 페이지와 자연스럽게 연결하되, 현재 페이지의 중심 장면을 유지한다.

본문 작성 규칙:

- 현재 페이지 본문만 작성한다.
- 한 페이지에는 하나의 중심 장면이나 중심 감정만 담는다.
- sceneSummary에 없는 큰 사건을 임의로 추가하지 않는다.
- 주인공의 행동, 감정, 주변 장면이 자연스럽게 들어가야 한다.
- mainEmotion이 본문 안에서 느껴지게 작성한다.
- 독자 나이 규칙에 맞는 문장 난이도와 어휘를 사용한다.
- 동화 장르 규칙에 맞는 따뜻하고 안전한 분위기를 유지한다.
- bodyText는 항상 해요체로 끝낸다(예: "~했어요.", "~있어요.", "~였어요.", "~할까요?").
- "~습니다/~했습니다" 같은 격식체나 "~다/~였다" 같은 평서체(서술체)는 쓰지 않는다.
- 안전 규칙을 따른다.
- 잔혹한 장면, 과도한 공포, 생명 위협, 심한 폭력은 만들지 않는다.
- 문제 해결은 관찰, 대화, 협력, 용기, 정직, 다시 시도하기 중심으로 진행한다.
- 교훈을 직접 설명하지 않고 주인공의 행동과 감정 변화로 보여준다.

글자 수 기준 (중요, 절대 기준):

- bodyText는 공백 포함 최대 80자를 넘지 않는다. 이는 뷰어 화면의 실제 텍스트 박스 크기(가로 400px, 세로 150px, 글자 크기 20px, 줄간격 1.6배 기준 약 4줄 × 20자)에서 계산된 물리적 한도이며, 독자 나이와 무관하게 항상 지켜야 하는 절대 상한이다.
- 80자를 넘기면 화면에서 잘려서 안 보이므로, 어떤 경우에도 80자를 초과하지 않는다.
- 보통 짧은 문장 1~2개 정도의 분량이다.
- 한 페이지에는 하나의 장면, 하나의 핵심 동작이나 대사만 담는다. 여러 사건을 욱여넣지 않는다.
- 글자 수가 부족한 것보다 넘치는 것이 훨씬 나쁘다 — 애매하면 더 짧게 쓴다.
- 문장 수보다 80자 제한과 현재 페이지의 장면 완성도를 우선한다.

bodyTextEn 작성 규칙:

- bodyTextEn은 bodyText와 같은 내용을 자연스러운 영어로 옮긴 번역이다.
- bodyText에 없는 사건이나 설명을 추가하지 않는다.
- 아동 그림책에 어울리는 쉽고 자연스러운 영어 문장으로 작성한다.
- 글자 수 제한은 bodyText 기준을 따르며, bodyTextEn 자체에는 별도의 글자 수 상한을 두지 않는다.

연속성 규칙:

- 이미 작성된 이전 페이지가 있으면 자연스럽게 이어지도록 한다.
- 주인공 이름, 성격, 배경, 중요한 물건, 문제 상황을 일관되게 유지한다.
- 이전 페이지에서 해결되지 않은 문제를 갑자기 없애지 않는다.
- 현재 페이지가 마지막 페이지가 아니라면 모든 문제를 완전히 해결하지 않는다.
- 현재 페이지가 마지막 페이지라면 전체 이야기를 안정적으로 마무리한다.
- 1페이지는 주인공과 배경을 자연스럽게 소개할 수 있다.
- 중간 페이지는 문제 해결을 위한 시도와 감정 변화를 보여준다.
- 마지막 페이지는 문제 해결, 회복, 성장, 따뜻한 마무리 중 하나 이상이 느껴지게 작성한다.

사용자 입력 반영 규칙:

- extra.userInput이 있으면 현재 페이지 본문에 반영한다.
- 사용자의 입력은 현재 페이지의 중심 문장, 행동, 감정, 대사 중 하나로 활용할 수 있다.
- 사용자의 고유명사, 장소, 물건, 핵심 사건은 최대한 유지한다.
- 사용자의 문장이 어색하면 독자 나이에 맞게 자연스럽게 다듬는다.
- 사용자의 입력을 완전히 다른 방향으로 바꾸지 않는다.
- 사용자 입력이 현재 페이지 계획과 크게 충돌하면 page plan을 우선하되, 가능한 범위에서 사용자의 의도를 반영한다.
- 사용자 입력이 안전 규칙이나 장르 규칙에 맞지 않으면 부드럽게 조정한다.
- 조정이 필요한 경우 warnings에 짧게 이유를 남긴다.

대사 사용 규칙:

- 대사는 사용할 수 있다.
- 대사는 너무 많지 않게 사용한다.
- 대사는 인물의 감정이나 장면 진행에 필요한 경우에만 사용한다.
- 독자 나이에 맞는 쉬운 말투를 사용한다.
- 대사만으로 페이지가 구성되지 않도록 한다.
- 말풍선이나 이미지 안의 글자를 지시하지 않는다.

summary 작성 규칙:

- summary는 현재 페이지의 내용을 1문장으로 요약한다.
- summary는 다음 페이지 작성 시 연속성 참고용으로 사용할 수 있어야 한다.
- summary에는 현재 페이지에서 실제로 일어난 핵심 사건만 넣는다.
- 다음 페이지의 사건을 summary에 미리 넣지 않는다.

nextQuestion 작성 규칙:

- nextQuestion은 다음 페이지로 자연스럽게 이어지도록 돕는 짧은 질문이다.
- 질문은 현재 장면과 직접 연결되어야 한다.
- 질문은 독자 나이 규칙에 맞는 난이도로 작성한다.
- 질문은 한 번에 하나만 묻는다.
- 사용자가 짧게 답할 수 있게 작성한다.
- 막연한 질문보다 행동, 감정, 장소, 물건, 선택 중 하나를 구체적으로 묻는다.
- 현재 페이지가 마지막 페이지라면 nextQuestion은 빈 문자열 ""로 둔다.

출력 규칙:

- 공통 응답 envelope는 common/json_rule.txt를 따른다.
- status가 "SUCCESS"인 경우 result에는 반드시 pageNo, sceneTitle, bodyText, bodyTextEn, summary, nextQuestion을 넣는다.
- pageNo는 extra.currentPageNo와 같아야 한다.
- sceneTitle은 현재 page plan의 sceneTitle을 유지하거나 자연스럽게 다듬는다.
- bodyText에는 현재 페이지 본문만 넣는다.
- bodyText 안에 페이지 번호, 제목 설명, JSON 설명, 마크다운을 넣지 않는다.
- bodyTextEn에는 bodyText를 자연스럽게 번역한 영어 본문만 넣는다.
- 실제 이미지 프롬프트는 생성하지 않는다.
- JSON key는 camelCase를 사용한다.
- 사용자에게 보이는 message는 한국어로 작성한다.

result 구조:
{
"pageNo": 1,
"sceneTitle": "",
"bodyText": "",
"bodyTextEn": "",
"summary": "",
"nextQuestion": ""
}

성공 응답 예시:
{
"status": "SUCCESS",
"taskType": "WRITE_PAGE",
"message": "현재 페이지 본문이 완성되었어요.",
"result": {

"sceneTitle": "루루가 문을 여는 순간",

},

}
