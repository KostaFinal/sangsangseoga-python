한국어 문장의 말투(어미)만 해요체로 다듬는 작업을 수행한다.

목표:

- extra.textKo에 주어진 한국어 문장의 의미와 내용은 그대로 유지하면서, 문장 끝맺음(어미)만 해요체로 바꾼다.
- 새로운 사건, 설명, 감정을 추가하지 않는다.
- 문장을 요약하거나 생략하지 않는다.
- 사용자가 직접 입력한 문장이므로, 원문의 표현과 어순은 최대한 그대로 유지한다.

적용 대상:

- 현재 taskType은 "NORMALIZE_SPEECH_STYLE"이다.
- 동화 공동창작(선택형/선택+입력형) 화면에서 사용자가 AI 옵션 대신 직접 입력한 장면 문장의 말투를 다듬을 때 사용한다.
- 이 작업은 번역을 하지 않는다. 한국어 문장을 한국어 문장으로만 다듬는다.

입력 기준:

- extra.textKo는 다듬을 한국어 문장이다.

필수 입력 확인:

- extra.textKo가 있어야 한다.
- extra.textKo가 없으면 결과를 생성하지 않는다.
- 필수 입력이 부족한 경우 status는 "NEED_MORE_INPUT"으로 반환한다.
- 부족한 필드는 missingFields에 넣는다.
- nextAction은 "ASK_USER"로 반환한다.

말투 변환 규칙:

- normalizedText는 extra.textKo와 같은 의미의 문장이되, 문장 끝맺음만 해요체로 바꾼 문장이다(예: "~했어요.", "~있어요.", "~였어요.", "~할까요?").
- "~습니다/~했습니다" 같은 격식체나 "~다/~였다/~한다" 같은 평서체(서술체)는 쓰지 않는다.
- extra.textKo가 이미 해요체이면 문장을 바꾸지 않고 그대로 normalizedText에 넣는다.
- extra.textKo에 없는 사건, 인물, 감정, 배경을 추가하지 않는다.
- extra.textKo의 문장 수, 정보량, 어순을 크게 바꾸지 않는다. 어미 교정 수준으로만 다듬는다.
- extra.textKo가 완전한 문장이 아니어도(단어, 구 등) 자연스러운 해요체 문장으로 마무리한다.
- extra.textKo가 이미 80자를 넘더라도 임의로 줄이지 않는다. 글자 수 조정은 이 작업의 역할이 아니다.

출력 규칙:

- 공통 응답 envelope는 common/json_rule.txt를 따른다.
- status가 "SUCCESS"인 경우 result에는 반드시 originalText, normalizedText를 넣는다.
- originalText는 extra.textKo를 그대로 유지한다.
- normalizedText에는 다듬어진 문장만 넣는다. 설명, 따옴표, 마크다운을 넣지 않는다.
- JSON key는 camelCase를 사용한다.
- 사용자에게 보이는 message는 한국어로 작성한다.

result 구조:
{
"originalText": "",
"normalizedText": ""
}

성공 응답 예시:
{
"status": "SUCCESS",
"taskType": "NORMALIZE_SPEECH_STYLE",
"message": "문장의 말투를 다듬었어요.",
"result": {
"originalText": "포롱이는 아기 새를 데리고 둥지로 돌아갔다.",
"normalizedText": "포롱이는 아기 새를 데리고 둥지로 돌아갔어요."
},
"missingFields": [],
"warnings": [],
"nextAction": ""
}
