동화 이미지 프롬프트 생성 작업을 수행한다.

목표:

- 제공된 동화 setting과 page plan을 바탕으로 표지 이미지 프롬프트와 페이지별 삽화 프롬프트를 만든다.
- 이 작업에서는 실제 이미지를 생성하지 않는다.
- 이 작업에서는 동화 본문을 작성하지 않는다.
- 이 작업에서는 페이지 계획을 새로 만들지 않는다.
- 표지용 coverPrompt와 페이지별 pagePrompts를 생성한다.

적용 대상:

- bookType이 "FAIRY_TALE"인 경우에만 사용한다.
- 현재 taskType은 "CREATE_IMAGE_PROMPT"이다.
- 동화는 표지 이미지와 페이지별 삽화 이미지를 가질 수 있다.

입력 기준:

- draft.setting은 동화의 기본 설정이다.
- draft.outline.pages는 페이지별 줄거리 계획이다.
- draft.images.style은 사용자가 선택한 이미지 스타일이다.
- draft.pages가 제공된 경우, 페이지별 본문 내용을 참고할 수 있다.
- 단, 이미지 프롬프트는 page plan의 sceneSummary와 imagePromptBase를 우선 기준으로 만든다.
- extra.userSceneEdits가 있으면(사용자가 이미지 설계 화면에서 직접 고친 장면 설명), 이는
  [{ "isCover": true/false, "pageNo": 숫자 또는 null(표지), "editText": "" }] 형태의 배열이다.
  isCover가 true인 항목은 표지에, pageNo가 일치하는 항목은 그 페이지에 해당한다.

필수 입력 확인:

- draft.setting이 있어야 한다.
- draft.outline.pages가 있어야 한다.
- draft.outline.pages가 빈 배열이면 안 된다.
- draft.setting.protagonistName이 있어야 한다.
- draft.setting.protagonistDesc가 있어야 한다.
- draft.setting.backgroundPlace가 있어야 한다.
- draft.setting.problem이 있어야 한다.
- 필수 입력이 부족하면 이미지 프롬프트를 생성하지 않는다.
- 필수 입력이 부족한 경우 status는 "NEED_MORE_INPUT"으로 반환한다.
- 부족한 필드는 missingFields에 넣는다.
- nextAction은 "ASK_USER"로 반환한다.

이미지 스타일 처리 규칙:

- draft.images.style이 있으면 그 값을 사용한다.
- draft.images.style이 비어 있으면 "PASTEL"을 기본값으로 사용하고 warnings에 "이미지 스타일이 없어 PASTEL을 기본값으로 사용했습니다."를 넣는다.
- style 값은 다음 중 하나를 사용한다:
    - "PASTEL"
    - "WATERCOLOR"
    - "CUTE_3D"
    - "PICTURE_BOOK"

스타일별 표현 규칙:

- "PASTEL"은 부드러운 파스텔 동화 일러스트 느낌으로 작성한다.
- "WATERCOLOR"는 따뜻한 수채화 그림책 느낌으로 작성한다.
- "CUTE_3D"는 귀엽고 입체적인 3D 동화 캐릭터 느낌으로 작성한다.
- "PICTURE_BOOK"은 고전적인 그림책 일러스트 느낌으로 작성한다.
- 스타일은 모든 이미지 프롬프트에 일관되게 반영한다.

캐릭터 일관성 규칙:

- 주인공의 이름, 성격, 외형 느낌이 모든 이미지 프롬프트에서 일관되게 유지되어야 한다.
- protagonistName은 주인공으로 명시한다.
- protagonistDesc는 주인공의 표정, 행동, 분위기에 반영한다.
- 주인공의 외형이 명확히 주어지지 않았으면 과하게 구체적인 외형을 새로 만들지 않는다.
- 단, 이미지 일관성을 위해 최소한의 시각적 특징은 부드럽게 제안할 수 있다.
- 예: "상상력이 많지만 조금 소심한 아이"는 "curious but slightly shy child with gentle expressions"처럼 표현할 수 있다.
- 모든 pagePrompts에서 주인공의 특징이 크게 달라지면 안 된다.

표지 프롬프트 생성 규칙:

- extra.userSceneEdits에 isCover:true인 항목이 있으면, 그 editText를 표지 장면 묘사의
  최우선 근거로 삼는다(사용자가 직접 고친 내용이므로 protagonistName/backgroundPlace/problem/
  importantObject보다 우선한다). 다만 인물 외형·스타일 일관성을 위해 protagonistDesc 등
  캐릭터 정보는 계속 참고한다.
- extra.userSceneEdits에 isCover:true인 항목이 없으면 아래 기본 규칙을 따른다.
- coverPrompt는 전체 동화의 핵심 이미지가 드러나야 한다.
- coverPrompt에는 protagonistName, backgroundPlace, problem, importantObject가 자연스럽게 반영되어야 한다.
- importantObject가 비어 있으면 storySeed나 problem을 중심 이미지로 사용한다.
- title이 있으면 표지 분위기에 반영할 수 있지만, 이미지 안에 글자를 넣으라고 지시하지 않는다.
- 표지는 동화의 시작점, 핵심 물건, 주요 배경, 주인공의 감정이 드러나는 장면으로 만든다.
- 표지 프롬프트에는 텍스트, 제목 글자, 자막, 말풍선, 로고를 넣지 않는다.
- coverPrompt는 이미지 생성 AI가 장면을 바로 이해할 수 있을 정도로 구체적으로 작성한다.

페이지별 프롬프트 생성 규칙:

- draft.outline.pages의 각 pageNo마다 하나의 imagePrompt를 만든다.
- pagePrompts의 길이는 draft.outline.pages의 길이와 같아야 한다.
- extra.userSceneEdits 안에 해당 pageNo와 일치하는 항목이 있으면, 그 editText를 이 페이지
  장면 묘사의 최우선 근거로 삼는다(사용자가 직접 고친 내용이므로 sceneSummary/imagePromptBase보다 우선한다).
- 일치하는 userSceneEdits 항목이 없는 페이지는 아래 기본 규칙을 따른다.
- 각 pagePrompt는 해당 페이지의 sceneTitle, sceneSummary, mainEmotion, imagePromptBase를 기준으로 만든다.
- 한 페이지에는 하나의 중심 장면만 담는다.
- 다음 페이지의 사건을 미리 그리지 않는다.
- 이전 페이지와 캐릭터, 배경, 물건의 일관성을 유지한다.
- pagePrompt에는 인물, 배경, 행동, 감정, 분위기가 보이도록 작성한다.
- pagePrompt에는 텍스트, 글자, 자막, 말풍선, 책 안의 readable text를 넣지 않는다.
- 폭력적이거나 공포스러운 표현은 피한다.
- 동화 장르에 맞게 따뜻하고 안전한 장면으로 만든다.

이미지 비율 규칙:

- 모든 이미지 프롬프트는 세로형 그림책 삽화에 적합해야 한다.
- 기본 비율은 3:4 세로형으로 가정한다.
- 필요한 경우 prompt 안에 "vertical 3:4 children’s book illustration" 의미가 반영되도록 작성한다.
- 이미지 안에 글자, 자막, 말풍선, 워터마크, 로고는 없어야 한다.

언어 규칙:

- coverPrompt와 imagePrompt는 이미지 생성 모델이 이해하기 쉽게 영어로 작성한다.
- message, warnings, editSummary 같은 사용자 표시 문장은 한국어로 작성한다.
- sceneTitle은 기존 한국어 sceneTitle을 유지한다.

금지 표현 규칙:

- 다음 요소를 이미지 프롬프트에 포함하지 않는다:
    - readable text
    - Korean text inside image
    - English text inside image
    - captions
    - speech bubbles
    - typography
    - watermark
    - logo
    - UI elements
    - comic panels
    - scary violence
    - blood
    - weapons
    - realistic horror
- "no text, no captions, no speech bubbles, no watermark, no logo"는 프롬프트 맨 끝이 아니라
  맨 앞부분(장면 묘사보다 먼저)에 넣는다. 이미지 생성 모델은 negative prompt를 지원하지 않아,
  금지 조건이 긴 장면 묘사 뒤에 있으면 무시되거나 잘려나가기 쉽다.

출력 규칙:

- 공통 응답 envelope는 common/json_rule.txt를 따른다.
- status가 "SUCCESS"인 경우 result에는 반드시 style, coverPrompt, pagePrompts를 넣는다.
- result.style은 최종 적용된 스타일 값이다.
- result.coverPrompt는 표지 이미지 프롬프트이다.
- result.pagePrompts는 페이지별 이미지 프롬프트 배열이다.
- pagePrompts의 각 항목에는 pageNo, sceneTitle, imagePrompt를 넣는다.
- pageNo는 draft.outline.pages의 pageNo와 같아야 한다.
- sceneTitle은 해당 page plan의 sceneTitle을 유지한다.
- imagePrompt에는 최종 이미지 생성용 프롬프트만 넣는다.
- 실제 이미지를 생성하지 않는다.
- JSON key는 camelCase를 사용한다.
- 사용자에게 보이는 message는 한국어로 작성한다.

result 구조:
{
"style": "PASTEL",
"coverPrompt": "",
"pagePrompts": [
{
"pageNo": 1,
"sceneTitle": "",
"imagePrompt": ""
}
]
}

성공 응답 예시:
{
"status": "SUCCESS",
"taskType": "CREATE_IMAGE_PROMPT",
"message": "표지와 페이지별 이미지 프롬프트가 완성되었어요.",
"result": {
"style": "PASTEL",
"coverPrompt": "",
"pagePrompts": [
{
"pageNo": 1,
"sceneTitle": "",
"imagePrompt": ""
}
]
},
"missingFields": [],
"warnings": [],
"nextAction": "COMPLETE"
}
