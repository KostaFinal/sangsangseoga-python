시 번역 작업을 수행한다.

목표:

- extra.textKo에 주어진 시 본문(또는 그 일부)을 자연스러운 영어 시로 옮긴다.
- 새로운 내용을 추가하거나 원문에 없는 이미지·사건을 만들지 않는다.
- 설명하듯 풀어쓰지 않고, 원문의 함축과 여운을 살린다.

적용 대상:

- 현재 taskType은 "TRANSLATE_POEM"이다.
- 시(POEM) 본문을 번역할 때 사용한다.
- translate_text.txt(공용 번역, 아동 그림책 어조 기준)와 달리, 시의 함축·리듬을 살리는
  것을 우선한다.

입력 기준:

- extra.textKo는 번역할 시 본문(원문 그대로, 줄바꿈 포함)이다.

필수 입력 확인:

- extra.textKo가 있어야 한다.
- extra.textKo가 없으면 번역을 생성하지 않는다.
- 필수 입력이 부족한 경우 status는 "NEED_MORE_INPUT"으로 반환한다.
- 부족한 필드는 missingFields에 넣는다.
- nextAction은 "ASK_USER"로 반환한다.

번역 규칙:

- textEn은 원문과 같은 행/연 구조(줄바꿈 위치)를 최대한 그대로 유지한다.
- 원문에 없는 사건, 감정, 설명을 덧붙이지 않는다.
- 원문의 일부를 생략하지 않는다.
- 아동 그림책처럼 쉽게 풀어쓰지 않고, 시적인 함축과 리듬을 살린 영어 표현을 쓴다.
- 직역으로 어색해지는 부분은 시적 표현이 자연스럽게 읽히도록 의역하되, 이미지와 정서는 원문을 벗어나지 않는다.
- 은유는 은유로, 감각적 이미지는 감각적 이미지로 옮긴다(설명으로 풀어내지 않는다).
- 원문이 산문시(문단 형태)면 영어도 문단 형태를 유지하고, 행 구분이 있는 시면 그 행 구분을 유지한다.

분량 조정 규칙 (재번역 요청 시):

- extra.tooLong이 true이면, 직전에 준 영어 번역이 이 시가 들어갈 리더 페이지 박스보다
  길어서 화면에서 잘려 보인다는 뜻이다.
- 이 경우 원문의 이미지·정서·의미는 그대로 유지하면서, 더 간결한 단어와 문장 구조로
  압축해서 다시 번역한다.
- 분량을 줄이기 위해 원문에 있는 핵심 이미지나 감정을 빼지 않는다 — 같은 뜻을 더 짧은
  영어 표현으로 옮기는 방향으로 압축한다(예: relative clause를 형용사구로, 반복되는
  수식어를 정리).
- extra.tooLong이 false이거나 없으면 이 규칙은 적용하지 않는다.

출력 규칙:

- 공통 응답 envelope는 common/json_rule.txt를 따른다.
- status가 "SUCCESS"인 경우 result에는 반드시 textKo, textEn을 넣는다.
- textKo는 extra.textKo를 그대로 유지한다.
- textEn에는 번역된 영어 시만 넣는다.
- textEn 안에 따옴표, 설명, 마크다운을 넣지 않는다.
- JSON key는 camelCase를 사용한다.
- 사용자에게 보이는 message는 한국어로 작성한다.

result 구조:
{
"textKo": "",
"textEn": ""
}

성공 응답 예시:
{
"status": "SUCCESS",
"taskType": "TRANSLATE_POEM",
"message": "시 번역이 완료되었어요.",
"result": {
"textKo": "창밖에 내려앉은 달빛에서\n나는 오래된 마음을 꺼내요",
"textEn": "From the moonlight settling outside my window,\nI draw out an old, quiet heart."
},
"missingFields": [],
"warnings": [],
"nextAction": ""
}
