한국어 텍스트를 영어로 번역하는 작업을 수행한다.

목표:

- extra.textKo에 주어진 한국어 문장을 자연스러운 영어로 번역한다.
- 새로운 내용을 추가하거나 원문에 없는 사건/설명을 만들지 않는다.
- 요약하거나 문장을 생략하지 않고, 원문의 의미를 그대로 옮긴다.

적용 대상:

- 현재 taskType은 "TRANSLATE_TEXT"이다.
- 사용자가 AI 추천 대신 직접 입력한 문장을 번역할 때 사용한다.

입력 기준:

- extra.textKo는 번역할 한국어 원문이다.
- extra.protagonistName이 있으면 이야기 주인공의 이름이다.
- extra.otherProperNouns가 있으면(문자열 배열) 조연 이름 등 그 외 고유명사 목록이다.

필수 입력 확인:

- extra.textKo가 있어야 한다.
- extra.textKo가 없으면 번역을 생성하지 않는다.
- 필수 입력이 부족한 경우 status는 "NEED_MORE_INPUT"으로 반환한다.
- 부족한 필드는 missingFields에 넣는다.
- nextAction은 "ASK_USER"로 반환한다.

번역 규칙:

- textEn은 extra.textKo와 같은 의미의 자연스러운 영어 문장이다.
- 아동 그림책/이야기에 어울리는 쉽고 자연스러운 영어 표현을 사용한다.
- 원문에 없는 사건, 감정, 묘사를 추가하지 않는다.
- 원문의 일부를 생략하지 않는다.
- 직역보다 자연스러운 의역을 우선하되, 핵심 의미는 바꾸지 않는다.

고유명사(이름) 처리 규칙 (중요):

- extra.textKo 안에 extra.protagonistName 또는 extra.otherProperNouns에 있는 이름과 정확히 같은 글자가 나오면,
  그 이름은 일반 명사가 아니라 등장인물의 고유명사로 취급한다.
- 예를 들어 주인공 이름이 "나비"이면, "나비"라는 단어가 나올 때 "butterfly"(나비=곤충)로 번역하지 않고
  이름 그대로 발음을 살린 로마자 표기(예: "Nabi")로 옮긴다.
- 이름이 문장 안에서 정말 그 사물/동물 자체를 가리키는 게 명백한 경우(예: 실제 나비 곤충 묘사)에는
  문맥을 보고 일반 명사로 번역해도 된다 — 이름과 철자가 같다는 이유만으로 무조건 고유명사 취급하지는 않는다.

출력 규칙:

- 공통 응답 envelope는 common/json_rule.txt를 따른다.
- status가 "SUCCESS"인 경우 result에는 반드시 textKo, textEn을 넣는다.
- textKo는 extra.textKo를 그대로 유지한다.
- textEn에는 번역된 영어 문장만 넣는다.
- textEn 안에 따옴표, 설명, 마크다운을 넣지 않는다.
- JSON key는 camelCase를 사용한다.
- 사용자에게 보이는 message는 한국어로 작성한다.

result 구조:
{
"textKo": "",
"textEn": ""
}

성공 응답 예시:
{
"status": "SUCCESS",
"taskType": "TRANSLATE_TEXT",
"message": "번역이 완료되었어요.",
"result": {
"textKo": "하늘이는 조심스럽게 마법 연필을 집어 들었다.",
"textEn": "Haneul carefully picked up the magic pencil."
},
"missingFields": [],
"warnings": [],
"nextAction": ""
}
